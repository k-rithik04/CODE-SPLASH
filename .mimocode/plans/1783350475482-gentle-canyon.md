# Fix: Low Image Quality on Team/Creators Cards

## Root Cause Analysis

There are **two compounding issues** causing low image render quality:

### Issue 1: `images.unoptimized: true` in `next.config.ts:38`

Next.js image optimization is globally disabled. This means:
- No automatic resizing to match display dimensions
- No automatic WebP/AVIF format negotiation
- The browser receives the **full raw image** and must scale it down via CSS alone
- The `quality={90}` prop on `<Image>` in `Team.tsx:57` has **zero effect** — it's ignored when `unoptimized` is true

This is likely set because `output: "standalone"` (line 31) doesn't include the Next.js image optimization server. However, this doesn't mean images must stay unoptimized.

### Issue 2: Canvas upload caps at 1200×1200 but no client-side optimization

`ImageUpload.tsx:132-142` caps canvas output at `maxW=1200, maxH=1200`. While 1200px is plenty for a ~320px card display, the image is served at full 1200px resolution with no server-side resizing. On mobile (280px card), the browser downloads and decodes a 1200px image to display at 280px — wasteful and can cause rendering artifacts on some browsers/devices.

### Issue 3 (Minor): Seed data images likely missing from Supabase Storage

Seed data (`schema.sql:502-506`) references `/assets/team/*.png` paths. There are **no files in `public/assets/team/`**. `getStorageUrl()` resolves these to Supabase Storage URLs like `{SUPABASE_URL}/storage/v1/object/public/cms-images/assets/team/naveen.png`. If these don't exist in the `cms-images` bucket, the error fallback icon renders instead.

## Recommended Fix

**Remove `unoptimized: true` and configure Next.js image optimization properly.** Since the project deploys with standalone output, we need to ensure the image optimization lambda is available, OR use Supabase Storage's built-in image transformations.

### Option A: Enable Next.js Image Optimization (Recommended)

This is the cleanest approach. Next.js standalone mode CAN serve optimized images if the deployment platform supports it (Vercel, Docker with the optimization lambda, etc.).

**File: `next.config.ts`**
1. Remove `unoptimized: true` (line 38)
2. Keep `remotePatterns` for Supabase hostnames
3. Optionally add `deviceSizes` and `imageSizes` for finer control

**File: `components/sections/Team.tsx`**
1. Remove the `quality={90}` prop (line 57) — let Next.js use its default of 75, which is excellent for photos
2. Ensure `sizes` attribute accurately reflects display size
3. Add explicit `width` and `height` for better layout (or keep `fill` with proper parent sizing)

### Option B: Use Supabase Storage Image Transformations

Supabase Storage supports on-the-fly image transformations via URL parameters:
```
/storage/v1/object/public/cms-images/team/photo.webp?width=640&quality=80
```

Modify `getStorageUrl()` to append transform parameters, ensuring images are always served at the right size.

### Option C: Upload at Target Size

Modify `ImageUpload.tsx` to accept a `maxSize` parameter and upload images at the actual display size (e.g., 640×640 for 2x retina on 320px cards).
